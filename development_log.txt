## 18. FDI 치아 번호 설계 오류 수정 및 구역 분류로 전환
- 발견 문제: 구현된 FDI 번호 매핑(#11~48) 알고리즘이 파노라마 이미지의 기하학적 복잡성(치아 배열의 비선형성, 정확한 교합면 위치 불명확)을 충분히 반영하지 못해 잘못된 번호를 출력하는 것을 확인.
- 원인 분석:
    - 교합면(Occlusal Plane)을 단순히 이미지 높이의 절반(h/2)으로 설정한 것이 부정확함.
    - 치아 위치를 정중선으로부터의 선형 거리로 매핑하는 방식이 실제 치아 배열 곡선과 불일치.
    - 정확한 FDI 번호 부여는 전용 치아 분할(Segmentation) 모델이 필요한 고난도 문제임.
- 조치 내용:
    - src/tooth_mapper.py를 전면 재작성하여 신뢰하기 어려운 정확 번호 대신 구역(Quadrant) 수준(상악 우측/좌측, 하악 우측/좌측)만 표시하도록 변경.
    - 교합면 보정: occlusal_y = img_h * 0.55 로 상단 보정 적용.
    - app.py의 시각화 및 결과 목록 출력을 구역 레이블 방식으로 교체 (예: 하악 좌측 (LL) Caries 0.85).
- 결론: 틀린 정보를 제공하는 것보다 신뢰 가능한 범위로 축소하는 것이 의료 AI로서 올바른 방향임.

## 19. 구역 분류(Quadrant) 기능 완전 제거
- 배경: 구역 수준 레이블도 파노라마 이미지 구조의 특성상 탐지 박스의 위치만으로는 신뢰도 높은 정보를 주기 어렵고, 탐지 결과의 가독성만 저하시킨다고 판단.
- 조치 내용: app.py에서 tooth_mapper 관련 임포트, 구역 분류 로직, 시각화 레이블을 모두 제거. 탐지 박스에는 클래스명과 신뢰도(Confidence Score)만 표시.
- 설계 원칙 확립: 도메인 지식 기반 메타데이터는 충분한 정확도가 보장될 때에만 추가한다.

## 21. 모드별 정량 성능 비교 결과 (Val Set 기준, 2026-05-15)
- 평가 대상: data/processed/val/images (학습에 사용하지 않은 검증용 이미지)
- 평가 스크립트: src/eval_mode_comparison.py
- 평가 조건: Confidence Threshold 0.25, IoU Threshold 0.4

비교 결과 (Caries 클래스 중심):

  클래스        | 지표       | Baseline | CLAHE  | 변화
  Caries        | Precision  | 0.338    | 0.323  | -0.015
  Caries        | Recall     | 0.495    | 0.515  | +0.020
  Caries        | F1         | 0.402    | 0.397  | -0.005
  Impacted      | Recall     | 0.925    | 0.900  | -0.025
  Periapical    | Recall     | 0.444    | 0.444  | 0.000
  Deep Caries   | Recall     | 0.531    | 0.469  | -0.062

- 해석:
    - CLAHE는 Caries Recall을 소폭(+0.020) 향상시키나 Impacted 및 Deep Caries에서는 오히려 하락. 고대비 경계를 가진 클래스에서 CLAHE가 과보정되는 것으로 추정.
    - 이전 세션에서 측정한 CLAHE 단독 Recall +114% 개선 수치는 임시 캐시 또는 측정 방식 차이에 의한 과장된 수치였음. 현재 결과가 더 신뢰도 높은 평가임.
    - SAHI 모드의 정량 평가는 CPU 기반 추론 시 이미지당 수 분 소요로 인해 Val 전체 평가가 현실적이지 않아 생략. GPU 환경에서 별도 실행 권장.
    - Caries Baseline Recall 0.495는 여전히 낮은 수치로, 데이터 정제 및 재학습 없이는 성능 한계에 도달한 상태로 판단.

2. Caries Recall 근본 개선: 데이터 정제 및 재학습
    - 현재 Recall 0.495는 모델 자체의 한계라기보다 라벨링 누락과 모호성에 기인할 가능성이 높음.
    - 실행 방법:
        a. src/analyze_errors.py로 FN(미탐지) 사례를 시각화하여 라벨링 누락 여부를 수동 검토.
        b. 특히 신뢰도가 0.25~0.40 사이인 탐지 결과 중 실제 충치일 가능성이 있는 박스를 라벨에 추가.
        c. 라벨 보완 후 yolo11s 기반으로 재학습 (runs/detect/dentex_yolov11s3 등 새 폴더에 저장).
        d. 재학습 후 eval_mode_comparison.py로 성능 재측정하여 개선 여부 정량 확인.

3. CLAHE 적용 범위 정교화
    - 현재는 모든 클래스에 동일한 CLAHE 설정을 적용하고 있으나, Caries에는 유효하고 Impacted에는 역효과가 나타남.
    - 개선 방향: 사용자가 클래스별 CLAHE 강도를 조절하거나, 탐지 목적에 따라 Caries 특화 / 전체 탐지 모드를 분리하는 방식을 검토.

4. XAI 히트맵 정확도 개선
    - 현재 Eigen-CAM은 이미지 가장자리 아티팩트를 완전히 제거하지 못하고 있음.
    - 다음 접근법을 순서대로 시도 권장:
        a. GradCAM++ 또는 LayerCAM으로 알고리즘 교체 (pytorch_grad_cam 라이브러리 내 대체 클래스 사용 가능).
        b. 탐지된 박스 영역(ROI)에만 히트맵을 마스킹하여 시각화하는 방식 도입 (ROI-Masked CAM).
        c. 최종적으로 개선이 어렵다면 XAI 기능 자체를 앱에서 제거하고 별도 분석 도구로만 유지.

5. SAHI GPU 성능 검증

    - CUDA 환경(RTX 4060)에서 eval_full_pipeline.py를 실행하여 SAHI의 실제 Recall 개선 수치를 측정.
    - 명령어: python src/eval_full_pipeline.py (use_sahi=True 설정 확인 후 실행)

6. 모델 경량화 및 추론 속도 최적화 (비판적 검토 3번 과제)
